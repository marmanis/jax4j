Iteration 6 · Benchmark

Where the GPU actually wins.

FP32 matrix multiplication in jax4j across four execution backends, on the same machine, at six sizes from 256² to 8192². Median of five timed runs after two warm-ups. The interesting shape is the crossover — small sizes are dispatch-dominated, large sizes are compute-dominated, and the ratio keeps growing.

56×
At 8192 × 8192, the RTX 4060 finishes in 4.65 s — 56× faster than the same code as Java parallel streams on all 16 CPU cores, which takes 4 minutes 22 seconds to do the same work.

Each CPU backend peaks where the working set still fits in cache and gets memory-bandwidth-bound after; the GPU peaks at 2048² and stays within a factor of ~1.4 across the rest. All numbers are a small fraction of theoretical hardware peak because TornadoVM's default triply-nested-loop kernel doesn't tile through shared/local memory — the RTX 4060's FP32 peak is ~11 000 GFLOPS (a well-tuned cuBLAS SGEMM lands close to that); the Ryzen 7845HX with AVX-512 FMA is ~1 500 GFLOPS. What we're seeing is the honest cost of a naive kernel across four dispatch paths, not the hardware ceiling.

Reading the shape

Small sizes are transfer-dominated

At 256², all three "parallel" backends lose to a single thread. The GPU pays a ~15 ms PCIe+dispatch floor per op; POCL pays even more (~34 ms) because each kernel is a fresh TornadoVM task graph. Below the crossover around 512², a plain Java loop is the fastest thing on the machine.

Java streams and POCL are indistinguishable

Both ultimately hit the same 16 cores. At 4096² they finish within 2% of each other (26.86 s vs 27.42 s). POCL's per-op task-graph overhead never quite amortizes for a workload this memory-bound — TornadoVM adds nothing over Java's Fork/Join here.

GPU speedup keeps growing

Matmul is O(n³) work over O(n²) data, so every doubling of n triples the compute per unit of transfer. The GPU line goes from 3.8× at 1024² to 39× at 2048² to 41× at 4096² to 56× at 8192². The 4060's 8 GB VRAM is only ¼ full at 8192² — the crossover keeps going.
